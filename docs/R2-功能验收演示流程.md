# R2 功能验收演示流程（课堂展示用）

> **适用版本**：当前图书管理系统（读者 / 馆员 / 管理员）  
> **R2 范围（与路线图对齐）**：读者续借、预约、借阅历史、图书详情；馆员现场借还（check-in）；管理员续借规则、逾期罚金、密码重置  
> **本版增量（版本 2 增强，课堂可并入 R2 演示）**：通过 **ISBN 拉取书目信息（Open Library）** 入库；每册生成 **图书馆专用条形码（LIB…，非 ISBN 条码）**；**打印标签**；工作台支持 **扫描馆码借还**

---

## 一、演示前准备（约 3 分钟）

| 序号 | 准备项 | 说明 |
|:---:|:---|:---|
| P1 | 服务启动 | 后端、前端已运行；数据库已迁移并完成种子数据（或测试账号可用） |
| P2 | 浏览器 | 建议使用无痕/双窗口：窗口 A 读者；窗口 B 馆员/管理员（或不同浏览器配置文件） |
| P3 | 测试账号 | 至少 1 名 **读者（MEMBER）**、1 名 **馆员**、1 名 **管理员**；记下邮箱与初始密码 |
| P4 | 图书数据 | 准备：**有库存** 书目 1 本（可借阅）；**无可用副本** 书目 1 本（用于预约演示，可先把副本借至 0） |
| P5 | 网络 | ISBN 查询需能访问 Open Library；课堂可先在家试通，现场备用 **已查好书目** 手填演示 |
| P6 | 浏览器打印 | 「打印标签」环节需允许弹窗/打印预览（或课前生成 PDF 备用） |

## 三、读者：图书详情 + 预约（Hold）

| 步骤 | 操作 | 预期结果 | 演示话术提示 |
|:---:|:---|:---|:---|
| 3.1 | 以读者登录，打开 **书目检索**，进入一本 **当前无可用副本** 的图书 **详情页** | 页面展示书名、作者、ISBN 等详情；显示不可借阅或可预约入口 | 「R2 要求读者能看完整图书详情，并在无库存时走预约。」 |
| 3.2 | 在详情页点击 **预约（Hold / Reserve）** | 提示预约成功或排队位置（如队列序号）；若已预约过同一本书，看到冲突提示 | 「预约对应路线图中的 reserve。」 |
| 3.3 | （可选）侧边栏进入 **My holds**，查看预约列表 | 列表中出现刚预约的图书记录 | 可快速带过，证明数据持久化。 |

---

## 四、读者：续借（Renew）

| 步骤 | 操作 | 预期结果 | 演示话术提示 |
|:---:|:---|:---|:---|
| 4.1 | 确保该读者有至少一笔 **借阅中且未逾期** 的记录（若无则先从详情页 **借阅** 一本有库存的书） | 「我的借阅」列表中出现 BORROWED 状态记录 | |
| 4.2 | 打开 **My loans（我的借阅）** | 未逾期行可操作；逾期行 **不应** 出现续借按钮（或续借被拒绝） | 「逾期不能续借，需到前台还书。」 |
| 4.3 | 点击 **Renew（续借）** | 应还日期向后延长；成功后列表刷新 | 「续借次数受系统规则限制，超过上限会报错。」 |
| 4.4 | （可选验收）重复点击续借直至达到 **最大续借次数** | 收到「已达最大续借次数」类提示 | 与管理员规则联动，留到 §7 也可一起说。 |

## 五、读者：借阅历史（Loan history）

| 步骤 | 操作 | 预期结果 | 演示话术提示 |
|:---:|:---|:---|:---|
| 5.1 | 侧边栏进入 **History（借阅历史）** | 列表按借阅时间展示多条记录（含已还与进行中） | 「区别于当前借阅，这里强调完整历史轨迹。」 |
| 5.2 | 指认列表中的 **罚金列/状态**（若曾有逾期归还） | 若有罚金配置且发生过逾期还书，可看到对应金额或状态 | 与 §6、§7 罚金规则呼应。 |
| 5.3 | （推荐）指认 **Library barcode** 列 | 新流程下每笔借阅可显示 **LIB… 册条码**（与 ISBN 不同） | 「一册一码，便于实物管理。」 |

## 六、馆员：现场还书（Physical return / check-in）

| 步骤 | 操作 | 预期结果 | 演示话术提示 |
|:---:|:---|:---|:---|
| 6.1 | 以 **馆员** 登录，进入 **Desk（前台 / Desk operations）** | 页面包含 **Check out** 与 **Return** 两个区域；文案强调优先 **馆码（LIB…）** | 「R2 馆员侧重点是现场办理借还。」 |
| 6.2 | 从读者「我的借阅」复制一笔 **借阅中的 loanId**，在 **Return** 的「Loan ID 或图书馆条码」中粘贴 **loanId**，提交 | 提示还书成功；图书可借副本 +1；若有册记录则该册状态回到在架 | |
| 6.2b | （与 §6A 联动）同一笔借阅改为在 **Return** 中粘贴 **该书副本的 LIB… 条码** | 仍成功还书（与 6.2 等价路径） | 「还书可以扫贴在书上的图书馆条码，不必只记 loanId。」 |
| 6.3 | **逾期罚金展示（与 R2 罚金规则联动）**：事先将某笔借阅的应还日改为过去（仅演示环境），再执行还书 | 成功归还，并提示 **逾期罚金金额**（若日罚金 > 0） | 「罚金由管理员配置的每日费率 × 逾期天数计算。」 |

## 六 A、馆员：ISBN 编目 + 馆藏书条码 + 打印标签（版本 2 增强）

**验收要点**：ISBN 只做 **查询元数据**；条码纸上印的是 **LIB… 馆码（Code 128）**，**不得**宣称「用 ISBN 条当馆藏条码」。

| 步骤 | 操作 | 预期结果 | 演示话术提示 |
|:---:|:---|:---|:---|
| 6A.1 | 馆员进入 **Add book**，输入 **正式出版 ISBN**，点击 **Look up ISBN** | 书名、作者等字段被自动填充（若 Open Library 无数据，可手填并说明依赖外网） | 「ISBN 用来对齐外部书目数据，减少手误。」 |
| 6A.2 | 填写 **Copies**（如 2），提交 **Add to catalog** | 提示成功；每条副本有独立 **libraryBarcode**；页面给出 **打印标签** 链接 | 「一种书目多册时，每册一个馆码，和 ISBN 一码多书不同。」 |
| 6A.3 | 打开 **Print barcodes** 标签页（或从 **Inventory** 进入「Print barcodes」） | 每册显示 **Code 128**，人眼可读值为 **LIB…** | 「这是馆里自己的资产编码，不是书的商品 ISBN 条码。」 |
| 6A.4 | 浏览器执行 **打印**，说明实际贴书流程 | 得到可剪切粘贴的标签（课堂可只展示预览） | 「纸质标签贴书脊或扉页，流通时扫馆码。」 |
| 6A.5 | 回到 **Desk → Check out**：读者邮箱填 `student1@library.local`（或课堂读者账号），**图书馆条码或 book ID** 填刚生成的 **LIB…** | 借出成功；读者 **My loans** 中该机读列出现同一 **LIB…** | 「外借链路绑定到『这一册』，不是只绑 ISBN。」 |

## 七、管理员：续借规则 + 逾期罚金

| 步骤 | 操作 | 预期结果 | 演示话术提示 |
|:---:|:---|:---|:---|
| 7.1 | 以 **管理员** 登录，进入 **Loan rules（借阅/规则配置）** | 可见 **MAX_RENEW_COUNT**、**FINE_RATE_PER_DAY** 等项 | |

| 7.3 | 修改 **每日罚金**，保存 | 保存成功；再测 §6.3，罚金金额随费率变化 | 「这是 overdue fee 配置。」 |

## 九、课堂展示用「一条龙」推荐顺序（约 15–20 分钟）

| 顺序 | 环节 | 用时参考 |
|:---:|:---|:---:|
| 1 | 准备（P1–P6） | 边讲边开 |
| 2 | §7 先改规则（续借次数、罚金）— 便于后面一举验证 | 2 min |
| 3 | **§6A ISBN 入库 + 打印馆码 + 扫馆码借出**（亮点，可放在读者流之前） | 4–5 min |
| 4 | §3 图书详情 + 预约 | 2 min |
| 5 | §4 续借 +（可选）用尽次数 | 2–3 min |
| 6 | §5 借阅历史（可指认 **馆码** 列） | 1 min |
| 7 | §6 现场还书：演示 **loanId** 与 **LIB 条码** 两种还书入口 + 逾期罚金 | 3 min |
| 8 | §8 密码重置并登录验证 | 2 min |
| 9 | Q&A | 剩余时间 |

**时间紧时**：可压缩 §4，或把 §6A 只做「查 ISBN → 保存 → 打开标签页」不打印。

## 十、验收结论（演示结尾可宣读）

- **读者**：图书详情、预约、续借、借阅历史均可在界面完成并符合业务规则；借阅记录可显示 **册级馆码（LIB…）**。  
- **馆员**：可通过 **loanId** 或 **图书馆条码** 完成现场还书；逾期时产生罚金提示（与配置一致）；可通过 **ISBN 查询** 编目、为每册生成 **非 ISBN 的馆码** 并 **打印标签**；可通过 **扫描馆码** 办理借出。  
- **管理员**：可维护续借规则与逾期日罚金，并可重置用户密码完成登录验证。  

以上内容覆盖 **R2 路线图**中的扩展能力，并包含 **本章新增：ISBN 辅助编目与馆藏书条码体系**；若课程要求对应 **用户故事编号**（如 R1.02、L1.02 等），请与任课教师提供的故事清单逐条打勾。
