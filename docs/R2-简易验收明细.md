# R2 + 条形码 · 简易验收明细

> 一眼打勾用；详细演示步骤见 `R2-功能验收演示流程.md`。

---

## 读者

| # | 验收点 | 通过（✓） |
|---|--------|-----------|
| 1 | 能打开图书**详情**（信息完整，无货时不能借） | |
| 2 | 无货时能**预约**，**My holds** 能看到 | |
| 3 | 有在借记录时能**续借**，**逾期**不能续借 | |
| 4 | **借阅历史**能看到多条记录 | |
| 5 | **My loans** 能显示 **Library barcode（LIB…）**（走完馆码借书后） | |

---

## 馆员

| # | 验收点 | 通过（✓） |
|---|--------|-----------|
| 1 | **Desk** 能办理借出（读者邮箱 + 书） | |
| 2 | **Desk** 能用 **loanId** 或 **LIB…** 办理**还书** | |
| 3 | 逾期还书有**罚金提示**（与管理员配置一致） | |
| 4 | **Add book**：**Look up ISBN** 能填元数据（或说明外网） | |
| 5 | 入库后每册有 **LIB…**，能打开**打印标签**（Code128，不是 ISBN 条码） | |
| 6 | 用 **LIB…** 办借出后，读者端能看到同一条码 | |

---

## 管理员

| # | 验收点 | 通过（✓） |
|---|--------|-----------|
| 1 | 能改**最大续借次数**，读者续借上限随之变 | |
| 2 | 能改**每日逾期罚金**，还书罚金随之变 | |
| 3 | 能**重置用户密码**，新密码能登录 | |

---

## 课前 5 项

| # | 准备 | 完成（✓） |
|---|------|-----------|
| 1 | `backend/.env` 有 `DATABASE_URL`，迁移 + seed 已做，前后端能启动 | |
| 2 | 三个账号可用：读者 / 馆员 / 管理员 | |
| 3 | 确认一本**可借**、一本**无货**（预约用） | |
| 4 | 教室网络试过 **ISBN lookup**，或准备手填入库备选 | |
| 5 | 打印/预览标签页无拦截（或备好截图） | |

---

**结论一句话**：R2 的读/管+现场借还+规则+重置密码都能走通；条形码侧 **ISBN 只编目、LIB 才是书条与扫描对象**。
