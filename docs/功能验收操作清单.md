# 功能验收操作清单（精确到账号与按钮）

统一说明：

- 打开站点：`http://localhost:5173`（需后端已启动）。
- 登录页：`/login`。顶部分别点 **Reader** / **Librarian** / **Admin** 标签，输入邮箱与密码后点 **Sign in**。
- 测试账号（与 `test_plan.md` 一致）：
  - 读者：`student1@library.local` / `student123`
  - 馆员：`librarian@library.local` / `librarian123`
  - 管理员：`admin@library.local` / `admin123`

---

## 1. 查看图书详情

1. 用 **读者** 账号登录（`/login` → 选 **Reader** → **Sign in**）。
2. 左侧点 **「Catalog search」**（或地址栏 `/books`）。
3. 在 **「Search the catalog」** 区域可设条件后点 **「Search」**，或直接浏览列表。
4. 在表格中**点击某一本书的书名链接**，进入详情页（`/books/:id`）。
5. 验收：可见书名、作者、ISBN、**Availability**、可选 **Description**、读者区 **Reader actions**（有副本时显示 **Borrow online** 等）。

---

## 2. 预约（Place hold）

**前置**：选一册 **当前无可借副本** 的书（种子数据里通常有「全部借出」的书目；或在演示前把某书借空）。

1. 用 **读者** 登录 → 左侧 **「Catalog search」**。
2. 检索并进入该书 **详情页**（点击书名）。
3. 在 **「Reader actions」** 中，若主按钮为 **「Unavailable」**，会出现 **「Place hold」**。
4. 点 **「Place hold」**。
5. （可选）左侧 **「My holds」** 打开列表，确认预约在列。

---

## 3. 续借（Renew）

**前置**：该读者至少一笔 **借阅中且未逾期** 记录（可先 **Borrow online** 一本有库存的书，或跑 `npm run demo:renew-fine` 准备演示数据）。

1. 用 **读者** 登录。
2. 左侧点 **「My loans」**（`/reader/loans`）。
3. 在未逾期、状态为在借的行，右侧点 **「Renew」** 按钮。
4. 验收：应还日期延后；若已达最大续借次数，出现错误提示。

---

## 4. 查看借阅记录（历史）

1. 用 **读者** 登录。
2. 左侧点 **「History」**（`/reader/history`）。
3. 验收：表格展示多笔记录（**Borrowed / Due / Returned / Fine / Status** 等）；有册条码时可在图书列看到 **LIB…**。

**说明**：当前在借列表在 **「My loans」**，历史在 **「History」**。

---

## 5. 图书管理员还书（含逾期罚金场景）

**前置（罚金）**：管理员在 **Loan rules** 中 **`Fine per day (reserved)`**（`FINE_RATE_PER_DAY`）大于 **0**；且存在一笔**已逾期**的在借记录（可用 `npm run demo:renew-fine`）。

1. 用 **馆员** 登录：`/login` → 选 **Librarian** → **Sign in**（默认进入 **Desk checkout**，`/librarian/desk`）。
2. 下翻到 **「Return」** 区域。
3. 在 **「Loan ID or library barcode」** 输入框粘贴：**借阅 ID** 或该书副本 **LIB…** 馆码（可先运行 `backend` 下 `npm run demo:renew-fine` 在终端查看 Loan ID）。
4. 点 **「Confirm return」**。
5. 验收：成功提示；若逾期且罚金大于 0，提示中含罚金金额。

**借书（可选）**：同页 **「Check out」** 填 **Patron email**、**Library barcode or book ID**，点 **「Confirm checkout」**。

---

## 6. 管理员：续借规则

1. 用 **管理员** 登录：`/login` → **Admin** → **Sign in**（默认进入 `/admin/users`）。
2. 左侧点 **「Loan rules」**（`/admin/rules`）。
3. 找到 **「Max renewals per loan (0 = disabled)」**（配置键 `MAX_RENEW_COUNT`），修改数值。
4. 点表单底部 **「Save」**。
5. 验收：提示 **「Saved.」**；读者再次 **Renew** 时上限行为与新值一致。

---

## 7. 逾期罚款（规则 + 还书联动）

1. **管理员** → **「Loan rules」** → 将 **「Fine per day (reserved)」**（`FINE_RATE_PER_DAY`）设为 **大于 0**（如 `0.50`）→ 点 **「Save」**。
2. 确保存在 **已逾期** 的在借记录（演示脚本 `cd backend && npm run demo:renew-fine` ，或手改数据库 `Loan.dueAt`）。
3. **馆员** → **「Desk checkout」** → **「Return」** → 填入该笔 **Loan ID** 或 **LIB…** → **「Confirm return」**。
4. 验收：还书成功提示中出现 **逾期罚金**（金额随天数 × 日罚金变化）。

---

## 8. 重置密码

1. 用 **管理员** 登录 → 左侧 **「Users」**（`/admin/users`）。
2. 在用户列表目标行右侧点 **「Reset password」**。
3. 页面下方绿色区域会显示 **「Password reset for …」** 及 **临时密码**（大号等宽字）。
4. 点 **「Sign out」** 注销，回到 `/login`。
5. 选 **Reader**（或该用户实际角色）标签，输入**该用户邮箱 + 临时密码**，点 **「Sign in」**。
6. 验收：能成功登录。
